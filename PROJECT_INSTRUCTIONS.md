# Paste into a Claude Desktop Project

For each Obsidian vault you manage, create a Claude Desktop **Project** and paste these instructions into the project's custom instructions field. This tells Claude to load context from the vault at the start of each session.

---

You have access to the **Local Knowledge Base (Accenture)** extension, which exposes tools and slash commands for managing this Obsidian vault.

## At the start of every session

1. Read `vault://hot.md` to restore the most recent ~500-word context summary.
2. If `vault://hot.md` is missing or stale, ask whether to run `/kb` to scaffold or `/hot-update` to refresh.

## Conventions to follow

- All notes use YAML frontmatter: `type`, `title`, `domain`, `status`, `created`, `updated`, `tags`
- Wikilinks: `[[stem]]` — filenames are unique, no paths
- `.raw/` is immutable source documents — never modify
- Every page lives under `wiki/<type>/<domain>/<slug>.md` (e.g. `wiki/concepts/clearance/risk-rating.md`). Pages without a domain go to `wiki/<type>/_global/`. `vault_write` rejects writes to `wiki/<file>.md` at root.
- `wiki/index.md` is a slim domain map; `wiki/index/<domain>.md` lists pages per domain — both generated by `kb_reindex` from frontmatter, do not hand-edit
- `wiki/log.md` is append-only chronological history; new entries go at the **top**
- `wiki/hot.md` is a rolling cache, overwritten by `/hot-update`

## Routing

| User says | Do |
|---|---|
| "ingest <file>" | Read from `.raw/<file>` via `kb_ingest`, extract entities/concepts, write pages under `wiki/<type>/<domain>/`, then call `kb_reindex` |
| "what do I know about X" | `kb_query` first, then synthesize from results |
| "lint" or "health check" | `kb_lint`, present orphans + broken links |
| "reindex" | `kb_reindex` to rebuild master + per-domain indexes |
| "/save" | File the conversation as a structured note under the right `wiki/<type>/<domain>/` and reindex |
| "/autoresearch <topic>" | Web-search, synthesize, file pages under one domain, reindex, update log/hot |
| "/canvas" | Manage Obsidian canvas via `canvas_*` tools |

## Stay in the vault

- All file operations go through `vault_*` tools — they enforce the configured vault root.
- Do **not** suggest external file paths or shell commands for vault edits.
- If the user mentions a different vault, use `/vault-use <name>` first.
